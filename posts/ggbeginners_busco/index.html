Making BUSCO pretty.

What is BUSCO?

Benchmarking sets of Universal Single-Copy Orthologs (or BUSCO) is a common method used to estimate the completeness and contamintation of a genome/transcriptome/proteome by counting a set number of genes that should only appear once in your dataset. See the user guide to give it a go yourself.

The pipeline produces a pretty decent plot but let’s make our own …

Setting up RStudio

I typically work in RStudio, particularly for data visualisation. I won’t go through the installation process here but it is pretty straighforward and described well here.

R is basically a collection of functions that manipulate data according to set parameters. Very clever and kind people write and organise these functions into packages, which we can install and load when needed.

We’re using the ggplot2 package for plotting but, as we’re also using dplyr ‘pipes’ for data formatting, we can load the tidyverse library here. The tidyverse is a whole thing. But generally, I prefer to use the “tidy data” approach. This is an excellent resource for all things R and all things tidy.

# Load the tidyverse package for data manipulation and plotting
library(tidyverse)

Data formatting

As of BUSCOv5.1.0, multiple fasta files can but analysed together in batch mode but here I am using outputs from individual analyses.

Let’s read in the busco data for each cell using the read_tsv function from the tidyverse package, skipping the first two rows of metadata.

read_tsv() produces tibbles whereas read.tsv() produces data frames. Learn more about tibbles here.

busco.cell1 <- read_tsv("~/Bioinformatics/scTranscriptomes/busco/1/full_table.tsv", skip=2)
busco.cell2 <- read_tsv("~/Bioinformatics/scTranscriptomes/busco/2/full_table.tsv", skip=2)
busco.cell3 <- read_tsv("~/Bioinformatics/scTranscriptomes/busco/3/full_table.tsv", skip=2)

Now we need to add a column to differentiate our cells so we can combine them into a single data frame.

busco.cell1$cell <- "cell1"
busco.cell2$cell <- "cell2"
busco.cell3$cell <- "cell3"

busco.df <- rbind(busco.cell1,
                  busco.cell2,
                  busco.cell3)

Using the piping from the dplyr package, we can now group the data by cell and BUSCO status, count the number of rows (and therefore genes) and then convert this value into a percentage.

busco.df.count <- busco.df %>%
  group_by(cell, Status) %>%
  tally() %>%
  mutate(percent = n / sum(n))

mutate() is a very useful tidyverse function allowing us to create new columns (in this case, one called “percent”). But it can also be used to modify existing columns or remove them.

In order to ensure the same value is always attributed to the same colour (particularly important when dealing with multiple plots involving the same data or groups),

I like to define colour palettes before hand. See my post on colours in plots for more details.

pal.BUSCO <- c(Complete = "#36AE7C",
               Duplicated = "#187498",
               Fragmented = "#F9D923",
               Missing = "#EB5353")

Before plotting, we can define the order of variable manually.

busco.df.count$Status.o <- factor(busco.df.count$Status,
                                  levels=c("Missing",
                                           "Fragmented",
                                           "Duplicated",
                                           "Complete"))

Now we can create a stacked bar chart showing the percentage of BUSCOs with each status for each cell.

ggplots are a series of layers. You start by defining the data, mapping variables to aesthetic characteristics of the plot (axes and fill in this case).

We then add the geometry layer, which determines the plot type. These are always associated with a statistical transformation and a positional argument, which are typically omitted due the default values giving a common desired effect. In this case, we want our bar plot to reflect the percentage values in our data frame (rather than a count of all occurences, for example)

ggplot(data=busco.df.count, aes(x=percent, 
                                y=cell, 
                                fill=Status.o))+
  geom_bar(stat="identity", 
           colour="black",
           width=0.75)+
  scale_y_discrete(expand=c(0,0.5))+
  scale_fill_manual(values=pal.BUSCO)

We can modify the appearance of a plot to our liking using the theme() layer. This has many options and most use the element_***() functions that you can see here.

element_blank() removes that variable altogethe (e.g., the grid lines corresponding to the major values of the y axis, panel.grid.major.y) The other common options are element_line(), element_rect(), element_border(), and element_text()

ggplot(data=busco.df.count, aes(x=percent, 
                                y=cell, 
                                fill=Status.o))+
  geom_bar(stat="identity", 
           colour="black",
           width=0.75)+
  scale_y_discrete(expand=c(0,0.5))+
  scale_fill_manual(values=pal.BUSCO)+
  theme(plot.margin=margin(4,1,4,1, unit="cm"),
        panel.grid.major.y = element_blank(), # Remove the y-axis grid lines
        panel.grid.major.x = element_line(colour="grey60",
                                          linetype="dashed"), # Add dashed x-axis grid lines
        axis.ticks = element_blank(), # Remove the y-axis ticks
        axis.title.y = element_blank(), # Remove the y-axis title
        axis.text = element_text(size=10), # Set the font size of the axis text
        axis.title = element_text(size=12, 
                                 face="bold"), # Set the font size and boldness of the axis title
        axis.text.y = element_text(margin = margin(r = -15)), # Move the y-axis text to the right
        panel.background = element_blank())+ # Remove the panel background
  xlab("% BUSCOs") # Add an x-axis label

Maybe you’re one of those people who have an irrational fear of bar charts, in which case, setting the geom_bar() position parameter to “dodge” will produced grouped bar charts.

In both plots, the scale layer can be used to …. modify the scale. Here we are just changing the distance between the limits of the axes and the border of the plot (purely, for aesthetic reasons) but we can also set a scale range here by using the limits=c(0,100) parameter.

However, modifying the limits this way will remove data beyond the specified limits. You will receive a helpful warning message telling you how many data points were removed. If using bar charts, this can remove the entire bar. To ‘zoom in’ on a specific range without losing data, we can instead use coord_cartesian().

# Create a dodged bar chart showing the percentage of BUSCOs with each status for each cell, and limit the x-axis range to 0-0.25
ggplot(data=busco.df.count, aes(x=percent, 
                                y=cell, 
                                fill=Status.o))+
  geom_bar(stat="identity", 
           colour="black",
           width=0.75,
           position="dodge")+ # Use the dodge position to create a dodged bar chart
  scale_y_discrete(expand=c(0,0.5))+
  scale_fill_manual(values=pal.BUSCO)+
  theme(plot.margin=margin(4,1,4,1, unit="cm"),
        panel.grid.major.y = element_blank(),
        panel.grid.major.x = element_line(colour="grey60",
                                          linetype="dashed"),
        axis.ticks = element_blank(),
        axis.title.y = element_blank(),
        axis.text = element_text(size=10),
        axis.title = element_text(size=12,
                                  face="bold"),
        axis.text.y = element_text(margin = margin(r = -15)),
        panel.background = element_blank())+
  xlab("% BUSCOs")+
  coord_cartesian(xlim=c(0,0.25)) # Limit the x-axis range to 0-0.25

Finally, we need to save our very pretty BUSCO plot. Then you can print it off and show your friends.

# Save the plot as a png file
ggsave(device="png", filename = "BUSCO_barplot.png",
        path="~/path/",
        dpi=320,
        height=7, width=7, units="in")
